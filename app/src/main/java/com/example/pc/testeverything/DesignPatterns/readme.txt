设计模式
观察者模式
我的业务需求是当去请求网络的时候如果发生网络异常，就通知网络管理去检查网络
但是由于请求网络部分的代码和网络管理分属于两个模块，网络管理模块依赖于网络部分。
在网络请求模块直接调网络管理中的方法是调不到的
所以决定采用观察者模式
观察者模式（Observer）完美的将观察者和被观察的对象分离开
RxJava的核心思想都是观察者模式
观察者模式与发布/订阅模式区别
https://www.cnblogs.com/lovesong/p/5272752.html
总结：观察者模式和发布订阅模式的区别应该就是当有房源消息的时候，到底是谁来通知租客，观察者是房东自己本人，而发布订阅则是中介。
